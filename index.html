Symbols в JavaScript

Техническая презентация для middle+ / senior разработчиков

1. Введение: зачем вообще Symbol

  • Проблема коллизий ключей в объектах
  • Ограничения строковых ключей
  • Исторический контекст (ES6)
const obj = { id: 1 };
obj.id = 2; // перезаписали

Подводка: А что если нужно гарантированно уникальное свойство?

2. Что такое Symbol

  • Примитивный тип (не объект!)
  • Уникальность по определению
  • Невозможность неявного преобразования в строку
const s1 = Symbol('id');
const s2 = Symbol('id');
console.log(s1 === s2); // false

3. Основные свойства Symbol

  • Уникальность
  • Не перечисляется в обычных циклах (for...in, Object.keys)
  • Используется как ключ объекта
const key = Symbol('secret');
const obj = {
  [key]: 123
};

4. Symbol как способ скрытия данных

  • "Псевдо-приватные" поля
  • Почему это не полноценный private
const secret = Symbol();
const user = {
  name: 'Aleksei',
  [secret]: 'hidden'
};

// Но:
Object.getOwnPropertySymbols(user); // можно достать

5. Глобальный реестр символов

  • Symbol.for
  • Symbol.keyFor
  • Когда нужен shared symbol
const s1 = Symbol.for('id');
const s2 = Symbol.for('id');
console.log(s1 === s2); // true

// Контраст:
// Symbol() → всегда новый
// Symbol.for() → из реестра

6. Встроенные символы (well-known symbols)

Ключевая часть доклада

  • Symbol.iterator
  • Symbol.toPrimitive
  • Symbol.toStringTag
  • Symbol.hasInstance
  • Symbol.species
  • Symbol.match / replace / search / split

7. Переопределение поведения языка

7.1 Symbol.iterator

const obj = {
  *[Symbol.iterator]() {
    yield 1;
    yield 2;
  }
};

7.2 Symbol.toPrimitive

Когда вызывается:

  • +obj
  • ${obj}
  • obj == 1
const obj = {
  [Symbol.toPrimitive](hint) {
    if (hint === 'number') return 42;
    return 'hello';
  }
};

7.3 Symbol.toStringTag

const obj = {
  [Symbol.toStringTag]: 'CustomObject'
};
Object.prototype.toString.call(obj);
// [object CustomObject]

8. Symbols в реальных библиотеках

  • React - использует $$typeof (Symbol) для определения элементов
  • Node.js - util.inspect.custom (Symbol)
  • Redux - internal action types

9. Когда использовать Symbol

✅ Хорошо подходит:

  • уникальные ключи
  • внутренние API
  • мета-программирование
  • расширение поведения языка

❌ Плохо подходит:

  • сериализация (JSON игнорирует)
  • публичные API
  • когда нужна простота

10. Подводные камни

  • ❗ Не видны в JSON.stringify
  • ❗ Не копируются через Object.assign (частично)
  • ❗ Трудно дебажить
  • ❗ Ложное ощущение приватности

11. Альтернативы

  • #private поля (классы)
  • WeakMap
  • обычные строки (если не нужна уникальность)

12. Итог

  • Symbol — это не просто "уникальный ключ"
  • Это инструмент для:
    • изоляции
    • расширения языка
    • контроля поведения объектов

13. Демонстрация (live coding)

  • создать iterable объект
  • переопределить toPrimitive
  • показать скрытые свойства
  • сравнить Symbol() vs Symbol.for()
