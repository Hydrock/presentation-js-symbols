# Слайд 14: Q&A / обсуждение

## Вопросы и обсуждение

### Когда вы бы использовали Symbol в проде?

- Для создания уникальных ключей в библиотеках
- Для скрытия внутренних данных в API
- Для расширения поведения встроенных объектов
- Для создания кастомных итераторов

### Использовали ли в кодовой базе?

- Внутренние символы в библиотеках
- Скрытие приватных данных
- Переопределение поведения объектов
- Создание специализированных интерфейсов

## Полезные практики

1. Используйте Symbol.for() для общих символов между модулями
2. Избегайте Symbol в публичных API
3. Документируйте использование символов
4. Учитывайте сложность отладки при использовании Symbol

## Рекомендации

- Используйте Symbol только тогда, когда действительно нужна уникальность
- Рассмотрите альтернативы (#private поля, WeakMap)
- Убедитесь, что команда понимает использование Symbol
