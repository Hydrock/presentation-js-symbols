# Слайд 9: Когда использовать Symbol

## ✅ Хорошо подходит:

### Уникальные ключи
Символы гарантируют уникальность, что полезно для ключей, которые могут конфликтовать.

```javascript
const key1 = Symbol('config');
const key2 = Symbol('config'); // Уникальный ключ
```

### Внутренние API
Для создания приватных свойств и методов внутри библиотек.

```javascript
const privateData = Symbol('private');
class MyClass {
  constructor() {
    this[privateData] = 'secret';
  }
}
```

### Мета-программирование
Для расширения поведения языка и создания кастомных интерфейсов.

### Расширение поведения языка
Для переопределения стандартных методов через well-known symbols.

## ❌ Плохо подходит:

### Сериализация (JSON игнорирует)
Символы не сериализуются в JSON.

```javascript
const obj = { [Symbol('key')]: 'value' };
console.log(JSON.stringify(obj)); // {}
```

### Публичные API
Для публичных интерфейсов лучше использовать обычные строки.

### Когда нужна простота
Символы усложняют код и могут быть непонятны другим разработчикам.
